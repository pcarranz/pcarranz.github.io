Hearing Challenge

 Find the code for this project on GitHub.


I was tasked with creating a hearing test challenge for the San Luis Obispo Children's Museum. The main goal was to design an exhibit that could teach kids that their hearing deteriorates over time.

As a result I created a button panel designed to test two people at once. This way a child can take the test along with their parent and realize that they have better hearing than them.

Functionality:
Two users will put on a set of headphones and press a 'Start' button when ready to begin the challenge. An Arduino with a wave shield will control a sequence of LEDs and sounds that both users will hear. Each sound is a varying frequency tone whithin the range of normal human hearing. If the the user hears the sound they will press a button next to the corresponding sound that will light up to keep a tally. At the end of the challenge each user can compare their results by counting the number of lit buttons each one has.

This was the first project that involved a significant amount of hardware to wire and control with an Arduino. At first, I had difficulties finding an efficient way to debounce the many buttons, 11 total, via the software as well as finding a way to trigger the button LEDs while the sound shield was in use.


My solution was to use an external library named, PinChangeInt, that essentially attaches an interrupt handler to each button pin and use hardware debouncing instead of software debouncing.

I used two 0.1 micro-farad capacitors to debounce the buttons which allowed the Arduino code to be simplified. The code for the wave shield is relatively long and very precise. Therefore, hardware debouncing allows for easier maintenance and better organization of the project overall.
